# PM-Core 架構文件索引地圖（INDEX）

> **用途**：本檔是三層文件的**總地圖**——`pm-core-architecture.md`（主檔＝現役 spec 單一真實來源）＋ `docs/archive/<功能群>.md`（已完成功能落地紀錄／退役草稿／歷史，`architecture-archive.md` 為 router）。
> **怎麼查**：先在本檔按功能群（左側 A–L 分節）找到 §編號 → 看「檔」欄 → `主`＝去主檔該節；`檔`＝去 **`docs/archive/<該群>.md`**（群＝本節所屬的 A–L）挖歷史細節。要查「某個已完成功能當初怎麼做的」→ 找到它 → 幾乎都在 archive 群檔。
>
> **狀態圖例**
> - 🟢 **現役** = 留主檔：原則／現役 spec／未做的設計／進行中／草稿。
> - 📦 **歸檔** = 進 archive 群檔：已完成功能的落地紀錄、退役草稿、施工順序等歷史沉澱。
> - 🗑 = 已刪除（過時／退役，git 史可追）。
>
> **「檔」欄**：`主`＝主檔｜`檔`＝archive 群檔（`docs/archive/<所屬 A–L 群>.md`）｜`主+檔`＝現役規格留主檔·落地紀錄搬 archive｜`刪`＝已刪。
> ✅ 步驟 1（索引）＋步驟 2（抽歷史瘦身）＋步驟 3（archive 按功能群分檔）已完成（2026-07-01）。行號欄為搬移前之值，僅供歷史對照；找 code 以函式名為錨（主檔 §18.7.2）。
