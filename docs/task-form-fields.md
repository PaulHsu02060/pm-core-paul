# 任務表單欄位規格（新增 / 編輯）

## 設計原則
- 必填只有：任務名、所屬專案。其餘全選填（資料缺損容忍）。
- 進階欄位「分組摺疊」，預設收起，降低新手恐懼、不犧牲專家能力。
- 專家欄位（FS/SS/FF/SF、里程碑、必須繳付、緊急程度）用 tooltip hover 解釋（data-tip，沿用 initTooltip），不佔版面、不擠在欄位下方。

## 基本欄位（攤平顯示）
- 任務名稱＊（必填）
- 說明
- 所屬專案＊：唯讀鎖定（在專案頁開單已綁定，不可選，不可能去A專案開B專案的單）
- 擔當：可手 key，系統累積記錄過的擔當→之後可下拉（動態收集）
- 緊急程度：系統自動算（依 deadline，緊急＝距完成日<2工作天）+ 允許手動覆蓋。各級用 tooltip 說明定義
- 狀態：唯讀顯示，由實際開始/完成日衍生（無實際開始＝未開始、有實際開始無完成＝進行中、有實際完成＝完成；延誤另由 deadline+今天算，不手選）
- 預計開始 / 預計完成（DEADLINE）：月曆下拉，可手改
- 預估工時（H）

## 移除的欄位（相對舊版）
- 分類：移除，改系統從工作內容自動判斷（連帶：行事曆配色改吃系統推斷值，列後續優化）
- 處理方式：移除

## 進階分組摺疊（四區）

### 1. 歸類
- PLM 階段：datalist（有資料專案可下拉、新專案打字建立，動態收集）
- 子群組：同上動態收集
- 類型：寫死三選一 + tooltip。任務（排甘特）/里程碑（節點，工期0，甘特畫菱形）/群組（母項，本身不執行）

### 2. 排程依賴（前置任務）
- 模糊搜尋：輸入單號或任務名的部分字 → 即時過濾下拉 → 點選加入（取代長清單，100筆也不怕）
- 每條前置：選關係類型（FS完成→開始預設/SS同時開始/FF同時完成/SF開始→完成，各帶tooltip白話）+ 延遲（工作天）
- 延遲欄位即時白話翻譯：0＝前置完成的次一個工作天開始；-1＝同天；2＝再隔2工作天（系統 FS lag=0 是隔一工作天，非同天，已查證）
- 可「加一條前置」對應多前置
- 不加前置＝無依賴任務，可獨立排程（簡易專案適用）

### 3. 交付與風險
- 勾「此任務有必須繳付的交付物」→ 才顯示繳付物說明欄
- 勾「此任務有風險議題/需關注」→ 才顯示風險文字欄
- 風險勾選後，再帶一個「需拉高層處理（HL）」次勾選 → 主頁總覽顯示

### 4. 實際執行（反向摺疊邏輯）
- 新增模式：預設收起（多數是規劃未來任務，還沒發生）
- 編輯既有任務：預設展開（單已開立、回填實際進度是常態）
- 欄位：實際開始 / 實際完成

## 與後端欄位對應
- 已存在欄位（同步路徑/資料模型在用，可直接讀寫）：stage / subgroup / predecessor / plannedStart / plannedEnd / actualStart / actualEnd
- M1 匯入器才會開始寫入的新欄位（表單 M2 實作前需確認已就緒）：mustDeliver / deliverable / riskIssue
- 結論：表單實作＝補前端輸入元件 + 讀寫欄位；其中交付/風險三欄依賴 M1 先把資料灌進來，M2 順序排在 M1 之後。
