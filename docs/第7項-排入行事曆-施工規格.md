## 改動清單（三處，逐一核 diff）

### ① task 加欄位 `scheduleToCalendar`

- 型別：布林，預設 `false`
- 存哪：跟其他 task 欄位一起，localStorage 既有結構，不另開 store
- 舊資料相容：既有 93 筆沒這欄，讀取時 `t.scheduleToCalendar === true` 對 undefined 自然是 false，**不需 migration**
- 新建任務（saveNewTask）：從表單勾選框讀值寫入
- 編輯任務（saveTask）：同上

### ② 表單加勾選框

- 位置：架構文件 §6.2 表格最後一列「排入行事曆 checkbox」
- 由 `App.buildTaskFormHtml(task, mode)`（app.js:3217 附近）產生
- **new/edit 共用同一份**（單一真實來源，禁複製兩份）
- edit 模式要回填現值（`task.scheduleToCalendar ? 'checked' : ''`）
- 加 `?` hover 說明：勾選後此任務會出現在總儀表板的時程表（視圖一），用於「我要親自排時間動手做」的任務

### ③ 分流函式 `getCalendarTasks`

```js
function getCalendarTasks(tasks) {
  return tasks.filter(t => t.scheduleToCalendar === true);
}
```

- 純函式：只回傳資料，不碰 DOM、不碰 Storage（守 §1 分層紀律）
- 放 core 區（跟 computeSchedule 等核心函式一起）
- **本次只到「函式存在且回傳正確子集」**，呼叫端（時程表 render）是第 8 項

## Commit-gate（嚴格遵守）

- 三處可拆成 2-3 個 commit（欄位+表單一筆、函式一筆），或合成一筆，看 diff 大小決定
- commit 前單獨 `git status`，確認只有預期檔、無機密檔（config.local.js / seed.local.js）
- `git add` 明確檔名，禁 -A / .
- commit 訊息兩段 -m，禁 Co-Authored-By，避 `>` `/`
- 家裡桌機跑過 56 測試 → **不標 [unverified]**（已驗證）
- `?v=` 版本號只升動到的檔（app.js 一定動；表單在 app.js 內，不一定動 index.html，視實際情況）
