# PM-Workspace UI/CSS 設計與排版規範

> 全系統視覺一致性的單一參考。任何 CSS 創建/調整前先讀這份，照表選 `:root` 變數與既有 class，
> 能少很多來回改動。設計調性：**深綠工程/產品開發風的暖色盤**（非 Tailwind 冷灰）。
>
> 來源：使用者統整的 UI/UX 規範（2026-06-26），已對應到本專案 `style.css` 既有 `:root` 變數。

---

## 0. 鐵則（最高優先，違反先停）

1. **顏色/圓角/z-index/陰影一律走 `:root` 變數**；hex 只准出現在 `:root` 定義處（見 `CLAUDE.md` CSS 鐵則）。
   合理例外：rgba 透明衍生、膠囊 99px、圓點 50%。
2. **新增顏色先查既有變數能否複用**，不能才開新的語意變數。
3. ⚠ **本規範裡標的 Tailwind 色票（如 `#15803d`）只是「設計意圖參考」，實作一律用右欄的本專案 `:root` 變數**，
   不要把 Tailwind hex 寫進 code，否則會多出第二套色盤、破壞單一真實來源。
4. 收斂寫死值做**等值重構**，不順手改色。
5. ⚠ 第一階段預覽頁 `.s1-preview` 目前帶 `zoom: 1.15`，**實際渲染 ≈ 設定 px × 1.15**。
   本表的 px 為「希望看到的最終大小」；在該頁設值時記得除以 1.15，或日後移除 zoom 後再校。

## 1. 字體層級（Typography Hierarchy）

嚴格區分「標題 / 控制項 / 輔助說明」三級，避免視覺重心混亂。

| 層級 | 用途 | 大小 | 粗細 | 顏色（本專案 token） | 既有 class / 備註 |
|---|---|---|---|---|---|
| 全站大標題 | 主頁/Modal 最上方主標題 | 24–28px | 700 | `var(--ink)` | `.s1-pagetitle` |
| 區塊大標題 | 如「階段區間預覽」獨立大區塊 | 16–18px | 600–700 | `var(--ink)` | `.s1-prev-title` |
| 卡片/欄位小標題 | 輸入框上方標題（專案名稱、選擇範本…） | 14px | 500 | `var(--ink2)` | `.s1-top-label` |
| 主要輸入/內容/膠囊 | 輸入框內文、下拉選中文字、流程膠囊 | 14px | 400 | `var(--ink)`／膠囊 `var(--ink2)` | inputs、`.s1-stage-chip` |
| 動態狀態區主旨 | 淺色提示區第一句開頭（正向排程中／時空警報） | 13–14px | 700 | 依狀態色（見 §2） | `.s1-dynhint`（主旨可加粗段） |
| 輔助/引導小字 | 頁/卡內所有大白話引導語、備註小字 | 12px | 400 | `var(--ink4)`（淡灰）/ `var(--ink3)` | `.s1-stage-note`、`.s1-tips-*` |

**輔助小字鐵則（核心防呆）**：引導/備註小字**絕不可用粗體、不可用全黑**——必須是 12px、淡灰（`--ink4`/`--ink3`），
讓它退為背景輔助線，焦點留給標題與控制項。

> 本專案文字色為**暖石色（stone）**非 Tailwind slate：
> `--ink`(#1A1612) / `--ink2`(#3A332A) / `--ink3`(#5A5346) / `--ink4`(#7E796D)。
> 對應意圖：Slate-800→`--ink`，Slate-600→`--ink2`，Slate-500→`--ink3`，Slate-400→`--ink4`。

## 2. 顏色與狀態（三色通用：大局燈號 + 甘特單階段）

三色語意一致：**綠=安全/可行/填開始日；黃=警告/偏緊/填上市日期；紅=延誤/不足/警報**。
**文字色必須明顯深於背景色**（符合無障礙對比），禁止過淡文字。

| 狀態 | 背景（提示區/膠囊底） | 文字/圖標 | 甘特長條 | 既有 class |
|---|---|---|---|---|
| 🟢 綠 安全/可行 | `--sage-50`（提示）/ `--sage-chip`（膠囊） | `--sage-700`～`--sage-800` | `--sage-500` | `.slack-pill-green`／`.s2-gbar-green`／`.sd-green` |
| 🟡 黃 警告/偏緊 | `--amber-l` | `--amber-ink` | `--amber` | `.slack-pill-yellow`／`.s2-gbar-yellow`／`.sd-yellow` |
| 🔴 紅 延誤/不足 | `--rose-l` | `--rose-ink` | `--rose` | `.slack-pill-red`／`.s2-gbar-red`／`.sd-red` |

> Tailwind 參考（僅意圖，**勿寫進 code**）：綠 50/700=#f0fdf4/#15803d、黃 100/700=#fef3c7/#b45309、紅 50/700=#fef2f2/#b91c1c。
> 本專案為**降彩度暖色**：sage(#4A7C5C)/amber(#C4956C)/rose(#B8504D)，照右欄 token 用即達同樣語意對比。

**標籤（教育卡情境 tag）對比**：底色維持淺、文字用同色系**深字**：
填開始日 `--sage-800`、填上市日期 `--terracotta-ink`、都填 `--navy-ink`。**禁用中間色當文字**（會糊）。

**品牌色**：Sidebar 深林綠、主要按鈕（深綠底白字、hover 加深）、次要/取消按鈕（淺灰底 `--stone-50`/灰字 `--ink3`/淡邊框）——
一律走既有按鈕 class 與 `:root`，勿新開色。

## 3. 佈局、間距與對齊（Layout & Alignment）

1. **頂部控制列垂直居中**：輸入框/色圈/下拉/文字同列時，父容器 `align-items: center;`，
   所有控制項中軸線對齊在同一水平線，嚴禁元件上下漂移。
2. **輸入框不強求滿版**：表單控制項保持適當寬度（如 `max-width: 300~320px`），靠左齊平；
   右側留白是正常的，嚴禁為填滿畫面把單一輸入框拉過長。
3. **卡片內膠囊（流程 badge）強制不斷行**：膠囊內距精簡（`padding: 2px 7px`～`2px 8px`）、字級固定 12px；
   寬度不夠時**縮字級/箭頭/間距或拉寬卡片**，**務必讓所有膠囊＋「＋」維持同一橫行**，
   嚴禁單顆膠囊被擠到第二行造成卡片高度不一。
4. **大區塊間距（呼吸感）**：主要大區塊之間（如頂部填寫區 ↔ 下方甘特預覽）垂直間距 `margin-top: 24~32px`，
   讓大局與預覽有明確分界。
5. **Icon 與文字間距**：提示區 icon 與後方文字 `gap: 8px`（必要時更大），**絕不可重疊或黏緊**。

## 4. 既有可複用 class / 變數速查（避免重造）

> 要做新元件前先看這裡能不能複用，能用就別新開。

- **燈號膠囊**：`.slack-pill` + `.slack-pill-{green|yellow|red}`；hover 提示 `.s1-pill-wrap`/`.s1-pill-tip`。
- **甘特列**：`.s2-gantt-axis` > `.s2-grow`（`.s2-gname` 名 / `.s2-gbar-track`>`.s2-gbar` 條 / `.s2-gdate` 日期）；
  條三色 `.s2-gbar-{green|yellow|red}`、空骨架 `.s2-gbar-none`。
- **案卡左框色**：`.case-card` + `.s2-case-main`（主 `--proj-c1`）/`.s2-case-other`（子 `--proj-c3`）。
- **HintBox（收折說明卡）**：`App.buildHintBox({key,title,summary,bodyHtml,collapsed,icon})`——
  自帶兩段式 tooltip 與展開持久化，**別另造說明卡/tooltip 引擎**。
- **動態提示**：`.s1-dynhint`（綠底）/`.s1-dynhint-init`（淡灰引導）/`.s1-dynhint-alert`（紅底警示）。
- **色點**：`.slack-dot` + `.sd-{green|yellow|red}`。
- **階段標籤膠囊**：`.stage-cap-pill` + `.cap-{0|1|2}`。
- **圓角/間距**：`--r4 --r6 --r8 --r10 --r12`；陰影 `--shadow-sm --shadow --shadow-lg`。

## 5. 調整後的總體視覺預期

標題層級分明、輔助小字退為淡灰不搶焦、三色狀態文字對比達標、控制列水平對齊、膠囊不斷行、
大區塊間距拉開——即達到「俐落、精緻、頂級 SaaS」的視覺質感。
